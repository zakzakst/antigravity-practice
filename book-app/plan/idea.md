# 本情報管理アプリ 仕様書案（アイデアメモ）

## 1. 概要
* **開発の目的**: 開発の練習目的
* **ターゲットユーザー**: 特定なし

## 2. 権限と機能ロール
本アプリにおけるユーザー種別と、それぞれの権限です。

1. **ゲスト（未ログイン）**
   * 本の検索、一覧表示、詳細情報の閲覧
2. **一般ユーザー（ログイン済）**
   * ゲストの全権限
   * 本のお気に入り登録 / 解除
   * 自身の読書記録・レビューの投稿 / 編集 / 削除
3. **管理者（Adminユーザー）**
   * ゲスト・一般ユーザーの全権限
   * システムへの「本」自体のデータ登録 / 編集 / 削除

## 3. コア機能要件
* **ユーザー機能**
  * ユーザー登録、ログイン、ログアウト
* **蔵書管理機能（Admin向け）**
  * 本の新規追加、編集、削除（タイトル、画像、著者などの情報）
* **閲覧・検索機能（全員向け）**
  * 登録されている本の一覧表示
  * 検索・フィルタリング機能（タイトル、著者、ジャンルなどで絞り込み）
* **ユーザーアクション機能（一般ユーザー向け）**
  * 本への「お気に入り」登録 / 解除
  * 本に対する「読書記録（感想・評価など）」の登録・表示

## 4. 画面ページ構成（案）
1. **トップページ（本一覧画面）** [全員]
   - 本のリスト表示、キーワード検索、絞り込み等
2. **本の詳細ページ** [全員]
   - タイトルやあらすじ、登録されているレビュー一覧
   - ※ログイン時のみ「お気に入りボタン」「レビュー投稿ボタン」を表示
3. **ユーザー登録・ログインページ** [ゲストのみ]
4. **マイページ** [ログインユーザーのみ]
   - 自分がお気に入り登録した本の一覧
   - 自分が投稿したレビュー等の履歴
5. **管理者画面（ダッシュボード）** [Adminのみ]
   - 新規本の登録フォーム、既存の「本データ」の編集/削除

## 5. データモデル（ERテーブル案）
* **users**: id, name, email, password, role(admin/user)
* **books**: id, title, author, description, cover_image_url, created_at
* **reviews**: id, book_id, user_id, rating, comment, created_at
* **favorites**: id, book_id, user_id, created_at

## 6. APIエンドポイント一覧（案）
必要な機能から逆算したREST APIの設計案です。

### ユーザー・認証関連 (Users & Auth)
* `POST /api/auth/register` : 新規ユーザー登録
* `POST /api/auth/login`    : ログイン
* `POST /api/auth/logout`   : ログアウト
* `GET    /api/users/me`    : ログインユーザーのプロフィール情報取得 [ログインユーザー]

### 本データ管理 (Books)
* `GET    /api/books`       : 本の一覧取得（検索クエリ `?q=` などを許容） [全員]
* `GET    /api/books/:id`   : 特定の本の詳細取得 [全員]
* `POST   /api/books`       : 本の新規登録 [Adminのみ]
* `PUT    /api/books/:id`   : 本の情報の更新 [Adminのみ]
* `DELETE /api/books/:id`   : 本の削除 [Adminのみ]

### 読書記録・レビュー (Reviews)
* `GET    /api/books/:id/reviews` : 特定の本のレビュー一覧取得 [全員]
* `GET    /api/users/me/reviews`  : 自分が投稿したレビュー履歴の一覧取得 (マイページ用) [ログインユーザー]
* `POST   /api/books/:id/reviews` : 対象の本へのレビュー投稿 [ログインユーザー]
* `PUT    /api/reviews/:id`       : 投稿済みレビューの編集 [投稿者本人のみ]
* `DELETE /api/reviews/:id`       : 投稿済みレビューの削除 [投稿者本人のみ/Admin]

### お気に入り (Favorites)
* `GET    /api/favorites`         : ログインユーザーのお気に入り本一覧取得 [ログインユーザー]
* `POST   /api/books/:id/favorite`: 対象の本をお気に入り登録 [ログインユーザー]
* `DELETE /api/books/:id/favorite`: 対象の本のお気に入り解除 [ログインユーザー]

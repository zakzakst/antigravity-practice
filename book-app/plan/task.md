# 実装タスク一覧 (Task List)

現時点で技術スタック（どの言語やフレームワークを使うか）が未定のため、一般的なWebアプリケーション開発の手順に沿って実装フェーズごとのタスクをリストアップしています。
後ほど技術スタックが決まり次第、具体的なコマンドやパッケージ名に落とし込んで行く予定です。

## フェーズ1: 開発環境の構築と初期設定
- [ ] バックエンド（APIサーバー）のプロジェクト作成・初期設定
- [ ] フロントエンド（UI）のプロジェクト作成・初期設定
- [ ] データベースのインストールおよび接続設定
- [ ] Linter / Formatter（コード成形ツール）の設定

## フェーズ2: データベース設計とモデル作成
- [ ] `users` テーブル（ユーザー情報と権限）のマイグレーションとモデル作成
- [ ] `books` テーブル（本データ管理）のマイグレーションとモデル作成
- [ ] `reviews` テーブル（読書記録）のマイグレーションとモデル作成
- [ ] `favorites` テーブル（お気に入り）のマイグレーションとモデル作成
- [ ] シードデータ（初期検証用のダミーデータ）の作成

## フェーズ3: バックエンド APIの実装 (※open-api.yaml 準拠)
- [ ] ルーティングの基本設定
- [ ] **ユーザー・認証 API** (Auth / Users) の実装
  - ユーザー登録処理・ログイン処理・トークン発行
  - ログインユーザー取得処理 (`/api/users/me`)
- [ ] **本管理 API** (Books) の実装
  - 本の一覧・詳細取得
  - 本の登録・編集・削除処理（Admin権限チェック追加）
- [ ] **レビュー API** (Reviews) の実装
  - レビュー一覧取得・投稿・編集・削除
  - 個人レビュー履歴取得 (`/api/users/me/reviews`)
- [ ] **お気に入り API** (Favorites) の実装
  - お気に入り登録・解除・自身のリスト取得

## フェーズ4: フロントエンド UIコンポーネントの実装
「モックUI」のデザインをベースに、実際のコンポーネントとして分割・実装します。
- [ ] ルーティング機構（画面遷移）の設定
- [ ] 共通レイアウト（Header 等）と共通コンポーネント（Button, Input 等）の作成
- [ ] **画面の実装** (静的モックの移植)
  - [ ] トップページ / 検索機能 UI
  - [ ] 本の詳細ページ UI
  - [ ] ログイン・ユーザー登録画面 UI
  - [ ] マイページ UI（タブ切り替え）
  - [ ] 管理者用ダッシュボード画面 UI（「本」の登録/編集フォーム）

## フェーズ5: APIとフロントエンドの結合（連携）
- [ ] ログイン機能の連携と認証状態のグローバル管理（State管理実装）
- [ ] APIから取得した「本の一覧・詳細」データのUI描画
- [ ] 本の検索処理とAPIの非同期通信の実装
- [ ] レビュー投稿フォームとAPIの連携
- [ ] お気に入り登録ボタンのアクション処理と状態同期
- [ ] マイページのユーザー情報・履歴・お気に入りリストの連携表示
- [ ] エラーハンドリング（通信失敗時やバリデーションエラー時の画面表示）

## フェーズ6: テスト・最終調整
- [ ] 管理者画面からのデータ登録・編集テスト
- [ ] 一般ユーザー画面でのアクション（お気に入り・レビュー）の結合テスト
- [ ] バリデーション（入力文字数や必須項目の制限）の挙動確認
- [ ] 見た目の微調整とパフォーマンス確認

# 書籍レビュー機能の実装

## Story
ユーザーは書籍に対して「星評価」と「テキストの感想」を投稿でき、また他ユーザーのレビューを一覧で閲覧できる。なぜなら、読書の記録を残すだけでなく、他の人の感想を読むことで本選びの参考にすることができるから。

## 要件
- **閲覧機能**: 本の詳細ページにて、対象の書籍に投稿された全レビューを一覧表示する。これは未ログインゲストを含む全員が閲覧可能。
- **投稿機能**: ログイン済みの一般ユーザーまたは管理者が「📝 レビューを書く」ボタンから新規レビュー（評価1〜5、感想コメント）を投稿できる。
- **編集・削除機能**: 自分が投稿したレビューに対してのみ、編集または削除を行える。
- **UI制限**: 未ログインユーザーに対しては投稿ボタンを非表示にするか、クリック時にログイン画面へ誘導する。

## UI設計 / モック情報
- `book-app/mock/detail.html` をベースに実装する。
- ページ上部アクションバーの「📝 レビューを書く」ボタンクリックで、投稿用フォーム（モーダルやアコーディオン展開などを想定）を表示するUIを新規に追加する。
- ページ下部の `.review-section` クラス部分に取得したレビューのリストを展開する。

## API連携仕様
- **一覧取得**:
  - `GET /api/books/{id}/reviews` を呼び出し、投稿者情報を含むレビュー配列を取得する（全ユーザー可）。
- **新規投稿**:
  - フォーム送信時に `POST /api/books/{id}/reviews` （Bearer Token必須）へリクエストを行う。
- **編集・削除**:
  - 自身のレビューに対して `PUT /api/reviews/{id}` または `DELETE /api/reviews/{id}` （Bearer Token必須）を実行する。

## 作業計画 (Implementation Plan)

### バックエンド (Backend)
1. **DB設計/マイグレーション**: 
   - `reviews` テーブル（id, book_id, user_id, rating, comment, created_at 等）を作成する。
2. **一覧取得 APIの実装**:
   - `GET /api/books/{id}/reviews`: レビュー情報に加えて、投稿者の名前等を取得できるように `users` テーブルをJOIN（またはリレーションを含める）して返却する。
3. **投稿・編集・削除 APIの実装**:
   - `POST /api/books/{id}/reviews`: バリデーション（`rating`が1〜5であること等）を実装し、DBへ保存する。
   - `PUT /api/reviews/{id}` / `DELETE /api/reviews/{id}`: 対象レビューの `user_id` と、実行者のトークン内 `user_id` が一致するか（認可）を必ずチェックし、処理を行う。
4. **評価の集計ロジック**:
   - （必要に応じて）レビューの追加・削除時に、`books` テーブル側の「平均評価」「レビュー件数」を同期・更新するバッチ処理またはトリガーを実装する（実行時に動的算出する場合は不要）。

### フロントエンド (Frontend)
1. **UIとステート(State)の定義**:
   - `reviews` (配列) と、フォーム開閉状態などのUI用ステートを定義する。
   - 初期表示時に `GET /api/books/{id}/reviews` をフェッチしてデータを格納する。
2. **レビュー一覧の描画**:
   - 取得したレビュー配列をループし、`mock/detail.html` の `.review-card` デザインをベースに動的に描画する。
3. **投稿フォームの実装**:
   - 星評価（Selectやラジオボタン等）とテキストエリアを持つフォームを作成する。
   - 認証状態を見てボタンの表示/非表示を切り替える。
4. **APIアクションの制御**:
   - 送信時、`POST` APIを呼び出し、成功したらステートに即時反映するかリストを再フェッチする。
   - 描画されたレビューが自分の投稿である場合のみ「編集」「削除」ボタンを表示し、対応するAPIを呼び出すロジックを組む。

# ToDoアプリ 実装計画

このドキュメントは、白を基調としたクリーンでモダンなデザインの、シンプルなブラウザベースToDoアプリの技術的な実装計画をまとめたものです。

## ユーザーレビューのお願い

> [!IMPORTANT]  
> ご要望の「削除確認ダイアログ」機能を追加しました！内容をご確認いただき、問題がなければ「承認します」または「OK」とご返信ください。コーディング作業を開始いたします。

## 提案する変更内容

アプリを軽量かつシンプルに保つため、標準のWeb技術（HTML、CSS、JavaScript）を使用して構築します。バックエンドサーバーや複雑なフレームワークは使用しません。

### フロントエンド アプリケーション

#### [NEW] [index.html](file:///c:/Users/e1433/OneDrive/デスクトップ/antigravity-practice/index.html)
- アプリケーションのメインとなる構造ファイルです。
- ヘッダー、新規タスクの入力欄、「追加」ボタン、およびタスク一覧を表示するリストコンテナを含みます。
- 「すべて」「未完了」「完了」をそれぞれ切り替えて表示するためのフィルター用ボタンを配置します。

#### [NEW] [style.css](file:///c:/Users/e1433/OneDrive/デスクトップ/antigravity-practice/style.css)
- 「白を基調としたクリーンでおしゃれな」デザインを実現するためのスタイル定義ファイルです。
- モダンなGoogle Fonts (例: Inter) や、やわらかな影（Box Shadow）、角丸（Border Radius）を活用し、上質で洗練されたカード型デザインに仕上げます。
- タスクの追加・完了・削除時にスムーズなアニメーション（マイクロアニメーション）を実装し、操作して気持ちの良いUIを目指します。

#### [NEW] [app.js](file:///c:/Users/e1433/OneDrive/デスクトップ/antigravity-practice/app.js)
- Vanilla JavaScript（フレームワークを使わない標準のJavaScript）を使用したアプリの基本ロジックです。
- タスクの作成（Create）、表示（Read）、更新（Update）、削除（Delete）の各機能を実装します。
- **[追加] 削除処理の直前に、「本当に削除してもよろしいですか？」という確認ダイアログ（ブラウザの標準の `confirm()` ダイアログ等）を出力します。「OK」が選ばれた場合のみ削除処理を実行します。**
- ブラウザの `localStorage` 機能を利用してタスクデータを保存・読み込みするロジックを記述し、ページをリロードしてもデータが消えないようにします。
- タスクのリストを描画したり、フィルターの状態に応じた表示の切り替え（DOM操作）を行います。

## 検証（テスト）計画

### 手動での動作確認
実装完了後、以下の手順でアプリが正しく動くかテストを行います。
1.  ブラウザで `index.html` を開く。
2.  新しいタスクを追加し、スムーズなアニメーションと共に表示されるか確認する。
3.  タスクを完了状態にし、取り消し線などの視覚的な変化が正しく付与されるか確認する。
4.  **タスクの削除ボタンを押し、確認ダイアログが表示されるかテストする。**
5.  **ダイアログで「キャンセル」を選んだ場合は削除されず、「OK」を選んだ場合は正しく一覧から消えるか確認する。**
6.  未完了・完了のフィルター機能が正しく動作するか確認する。
7.  ブラウザを更新（再読み込み）しても、タスクのデータが維持されているか確認する。
